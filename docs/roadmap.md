# Roadmap

Briefnexa is currently an MVP and portfolio case study. The source code remains private while the product is actively being developed.

The roadmap focuses on making the product more useful for recurring meetings, teams, and long-term meeting knowledge.

## Planned improvements

### Tasks dashboard

A dedicated dashboard for action items across meetings, with status, owners, due dates, and filtering.

### Cross-meeting search

Search across multiple meetings instead of only browsing or asking about one meeting at a time.

### Better speaker identification

Improve speaker separation and attribution so reports and action items can be tied to the right participants more clearly.

### User privacy settings for retention

Give users clearer control over how long transcripts, reports, and related meeting artifacts are retained.

### Team and workspace support

Support shared spaces for teams, projects, or organizations while keeping permissions clear.

### Calendar integrations

Connect meetings to calendar events so titles, attendees, dates, and context can be attached automatically.

### More export formats

Add export options beyond Markdown, such as PDF, DOCX, and structured task formats.

## Product direction

The long-term direction is to make Briefnexa a practical meeting memory layer: a place where users can capture meetings, understand what happened, find decisions later, and turn discussions into follow-up work.
